# Session Log: 2026-04-11 - Phase 7.6 - Filter Chip Regression Hotfix

## Overview
- **Date**: 2026-04-11
- **Phase**: 7 - 독자 레벨별 필터
- **Sub-task**: 7.6 - Filter chip click regression (hotfix)

## Problem Report

사용자 보고: "읽는 사람을 한번 선택하 바꾸고 싶어도 바꿀수가 업네."

첫 레벨 선택 이후 다른 레벨로 전환할 수 없음. 페이지 새로고침도 `localStorage`의
값이 재적용돼 동일 상태 복귀. 사용자가 첫 선택에 lock-in됨.

## Root Cause

`src/static/js/audience-filter.js` Line 57 (Phase 7.3 초기 구현):

```js
var cards = document.querySelectorAll('[data-audience]');
cards.forEach(function (card) {
    card.hidden = !matchesAudience(card, audience);
});
```

이 selector는 `data-audience` 속성을 가진 모든 요소를 매치한다. 그런데 필터 칩
자체(`<button class="audience-chip" data-audience="ALL">`, `GENERAL`, `DEVELOPER`,
`ML_EXPERT`)도 클릭 라우팅용으로 `data-audience` 속성을 갖는다.

사용자가 "개발자"를 클릭하면:
1. 매칭 article-card는 visible → 정상
2. "모두" 칩 (`data-audience="ALL"`) → `matchesAudience` false → `hidden`
3. "일반인" 칩 (`GENERAL`) → false → `hidden`
4. "개발자" 칩 (`DEVELOPER`) → true → visible (자기 자신만 남음)
5. "ML 전문가" 칩 (`ML_EXPERT`) → false → `hidden`

결과: **4개 중 1개 칩만 남아 사용자가 다른 레벨로 전환 불가**. 새로고침해도
`getSaved()`가 `DEVELOPER`를 반환해 `applyFilter('DEVELOPER')` 재실행 → 동일 결과.

## Fix

`[data-audience]` selector에 `:not(.audience-chip)` exclusion 추가. 상수화하여
한 곳에서 관리.

```js
var CARD_SELECTOR = '[data-audience]:not(.audience-chip)';
// ...
var cards = document.querySelectorAll(CARD_SELECTOR);
```

`applyFilter` 내 `.category-section` visible 체크에도 동일 exclusion 적용
(방어적). `.article-list` 내부 쿼리는 이미 `.article-card` 클래스로 scoping되어
있어 영향 없지만, 논리적 일관성을 위해 같은 selector로 정돈.

칩 조작 부분에는 `chip.hidden = false` 리셋 로직 추가 — 업그레이드 전 버전에서
localStorage에 저장된 비-ALL 값으로 접속한 사용자가 첫 init() 직후 모든 칩을
다시 보도록 보장 (defensive recovery).

## Why Tests Previously Missed This

Phase 7.1~7.5 구현 시 추가한 40개 테스트는 모두 Python (Enum, 직렬화,
`static_generator` 헬퍼) 레이어에 집중. **브라우저에서 JS가 DOM을 조작하는
런타임 동작은 테스트되지 않았다.** Pytest 환경에 jsdom/selenium 같은 JS 실행
인프라가 없으므로 실제 DOM 동작 검증은 어렵다.

대안으로 "JS 소스 파일을 정적으로 파싱해 알려진 함정 패턴을 금지"하는 방식의
regression guard를 추가했다. 완벽한 동작 검증은 아니지만 **이 특정 버그의 재발을
확실히 차단**한다.

## Potential Risks

1. **Pre-fix localStorage 값을 가진 사용자**: 배포 직후 첫 방문 시 기존 저장값
   (예: `DEVELOPER`)이 사용된다. `applyFilter`의 칩 반복에 `chip.hidden = false`
   리셋 라인이 있어 즉시 복구됨. ✅
2. **JS 캐시**: 일부 사용자 브라우저가 구 JS를 캐시할 수 있음. GitHub Pages의
   Cache-Control은 일반적으로 짧지만 강제 refresh 필요할 수 있음. **수용 가능.**
3. **회귀 테스트가 정적 분석 수준**: 실제 DOM 동작을 시뮬레이션하지 않음. 유사한
   다른 DOM 버그는 여전히 슬립 가능. **미래 개선 여지** (jsdom 도입) — 현재는
   수용.
