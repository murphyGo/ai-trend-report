## Key Decisions
| Decision | Rationale |
|----------|-----------|
| Blocklist(disabled) vs Allowlist(enabled) | 모든 소스가 기본 활성화라 disable만 나열하는 게 간결. 새 소스 추가 시 자동 활성화 |
| _COLLECTOR_REGISTRY 패턴 | Source enum → Class 매핑을 한 곳에서 관리. 수집기 추가/제거가 1줄 변경 |
| _DEFAULT_DISABLED 기본 비활성 | DEBT-001/002 소스를 하드코딩 주석 대신 데이터로 관리. 향후 해소 시 set에서 제거 |
| validate → validate_api_mode 리네임 | 이름이 범용처럼 보이지만 실제로는 API 모드 전용이라 혼란 방지 |
| validate_notifications 분리 | Slack 필수 요구가 Discord/Email만 쓰는 사용자를 차단하던 문제 해소 |
| 기존 validate() alias 유지 | 외부 스크립트가 validate() 호출할 수 있어 하위 호환 |
